# keiba_loop

競馬単勝回収率を自動改善する実験ループ。

## 目的

LightGBMモデルを使い、均等買い単勝の回収率100%超を目指す。

## 指標

| 指標 | 目標 | 説明 |
|------|------|------|
| **回収率** (主) | > 100% | 均等買い単勝 (100円) |
| **シャープレシオ** (副) | 高いほど良い | 月次回収率の安定性 |
| **的中率** (制約) | ≥ 10% | これを下回ると戦略として成立しない |
| **年間賭けレース数** (制約) | ≥ 600 | 統計的有意性の確保 |

## データ

- `Data_2020_fixed.csv`: 2020年〜2025年10月、約27.5万行
- 学習期間: 2020年〜2023年末
- テスト期間: 2024年〜
- **注意**: テスト期間のデータで学習しないこと（時系列リーク防止）

## 構成

| ファイル | 役割 | 編集可否 |
|----------|------|----------|
| `prepare.py` | データ読み込み・評価・可視化 | **変更禁止** |
| `train.py` | 特徴量・モデル・賭け戦略 | **ここだけ編集** |
| `results.tsv` | 実験結果ログ | 自動追記 |
| `progress.png` | 進捗グラフ | 自動生成 |

## Setup

1. `uv sync` で依存関係をインストール
2. `Data_2020_fixed.csv` がプロジェクトルートにあることを確認
3. `results.tsv` を初期化（最初の実行で自動作成される）

## 実験ループ

LOOP FOREVER:

1. **現状確認**: git state と results.tsv を確認
2. **仮説を立てる**: 特徴量、ハイパーパラメータ、賭け戦略のどれかを変える
3. **train.py を編集**: 仮説に基づいてコードを変更
4. **git commit**: 変更をコミット
5. **実験実行**: `uv run train.py > run.log 2>&1`
6. **結果確認**: `grep "^roi:\|^hit_rate:\|^sharpe_ratio:\|^total_bets:\|^constraints_met:" run.log`
7. **結果を記録**: results.tsv に追記
8. **判定**:
   - ROI が改善 → keep（ブランチを進める）
   - ROI が悪化 → discard（git reset --hard で戻す）
   - クラッシュ → crash（ログを確認して原因を特定）
9. **可視化更新**: `uv run -c "from prepare import plot_results; plot_results('results.tsv')"`
10. **次の実験へ**

## train.py で編集できるもの

### 特徴量エンジニアリング (`engineer_features`)
- カテゴリカル変数のエンコーディング方法
- 派生特徴量の追加（例: オッズの対数、騎手勝率、距離カテゴリ etc.）
- 外れ値処理、欠損値処理
- 特徴量選択

### モデル設定 (`LGBM_PARAMS`, `NUM_BOOST_ROUND`)
- LightGBM ハイパーパラメータ
- 学習ラウンド数
- Early stopping 設定

### 賭け戦略 (`select_bets`, 各種パラメータ)
- 閾値戦略: 予測確率が一定以上なら賭ける
- Top N 戦略: レースごとに上位N頭に賭ける
- オッズフィルター: 最小/最大オッズ
- 組み合わせ戦略の実装

## 制約事項

1. **prepare.py は変更しない**
2. **データリーク禁止**: `LEAKAGE_COLUMNS` に含まれるカラムを特徴量に使わない
3. **GPU不使用**: `device: "cpu"` のまま
4. **未来のデータで学習しない**: 時系列分割を変更しない

## results.tsv のフォーマット

```
experiment	roi	hit_rate	total_bets	annual_bets	sharpe	status	description	monthly_roi_json
```

status: `keep`, `discard`, `crash`

## 実験アイデアの方向性

### 特徴量系
- 時系オッズ情報の活用（指時系1〜4）
- 前走成績の深掘り（前走着順、前走脚質、前走PCI）
- 騎手・調教師のカテゴリ特徴量（コード化）
- 距離×馬場状態のクロス特徴量
- 休み明け戦目の活用
- 馬印・レース印の数値化

### モデル系
- ハイパーパラメータチューニング（num_leaves, max_depth, learning_rate）
- カスタム損失関数（回収率を直接最適化）
- アンダーサンプリング / オーバーサンプリング
- 複数モデルのアンサンブル

### 戦略系
- オッズフィルター最適化
- 期待値ベースの賭け選択（予測確率 × オッズ > 1）
- レース条件別の閾値調整
- Kelly criterion の適用

## NEVER STOP

実験ループが始まったら止まらない。人間が手動で中断するまで永遠にループし続ける。
アイデアが尽きたら、より大胆な仮説を試す。組み合わせを変える。前の失敗から学ぶ。
